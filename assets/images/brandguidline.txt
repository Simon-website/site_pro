## Identité

Nom       : WebCraft
Tagline   : Web Code, You Grow.
Baseline  : We code, you craft.
Logo      : assets/images/logo.png  (W stylisé avec < >, dégradé bleu→violet→rose)
Style     : Moderne · Punchy · Professionnel · Propre · Tech-friendly
Ambiance  : Dynamique et percutant, pas épuré à l'excès. Chaque section doit avoir du caractère.

---

## Direction artistique

L'objectif est un site "punchy" mais qui reste propre et moderne.
On évite le minimalisme fade : chaque section doit avoir une intention visuelle forte.
Points clés :
- Utiliser les dégradés généreusement (pas juste en accents)
- Créer du contraste entre sections (alterner fond clair / fond sombre)
- Les titres sont grands, gras (font-weight 700-800), percutants
- Les cartes ont toujours de la profondeur (ombres, hover lift)
- Le hero est split 2 colonnes sur desktop : texte à gauche, visuels flottants à droite

## Animations

Durées       : 0.25s (transitions UI) — 0.28s (reveal scroll)
Easing       : cubic-bezier(0.4, 0, 0.2, 1)
Cascade      : delay de 0.06s entre éléments d'une même grille
Hero entry   : fadeSlideUp 0.4s — chaque élément décalé de 0.12s
Scroll       : IntersectionObserver — classes .reveal / .reveal-left / .reveal-right
Compteurs    : animateCounter() déclenchée par IntersectionObserver sur .stat-number[data-target]

À éviter     : animations > 0.4s sur les éléments UI courants, effets lourds, canvas complexe

## Animations avancées (js/animations.js)

Curseur personnalisé
  Dot 6px bleu qui suit instantanément
  Ring 32px avec lag fluide (lerp 0.1) — grossit à 48px au survol des éléments interactifs
  Désactivé automatiquement sur mobile/tactile (matchMedia hover:hover)

Hero title word stagger
  Chaque mot du titre est wrappé en .hero-word via JS
  Animation fadeSlideUp 0.5s, délai 0.11s par mot
  Subtitle et actions décalés après le dernier mot

Hero cards lévitation
  Animation CSS heroFloat — translateY 0→-10px→0
  Durées différentes (4.2s / 5s / 4.6s) et délais (0 / 0.6s / 1.2s)
  Pause au hover + tilt 3D sur mousemove (perspective 800px)

Hero orbs parallax
  Scroll listener passive → translateY proportionnel à scrollY
  orb-1 : ×0.18 / orb-2 : ×-0.12 / orb-3 : ×0.10

Tilt magnétique
  addMagneticTilt() sur .service-card (6°), .why-small-card (5°), .stat-card (4°)
  perspective(900px) rotateY rotateX translateY(-5px) scale(1.01)
  Transition 0.35s ease au mouseleave pour retour fluide

Compteurs
  easeOut cubique (1 - (1-t)³) via requestAnimationFrame
  Durée 1600ms

Bouton shine
  Pseudo ::after qui traverse le bouton primary au hover (animation 0.55s)

Nav active
  IntersectionObserver sur les sections → classe .active sur le lien nav correspondant

## Panneau d'administration

URL d'accès    : admin/index.html
Mot de passe   : haché PBKDF2-SHA512 (100 000 itérations) + sel aléatoire côté SERVEUR
               → Pour changer : dashboard → Paramètres → Sécurité admin (min. 8 caractères)
               → Ne jamais modifier admin/js/admin.js pour changer le mot de passe
Authentification : JWT cookie HttpOnly + SameSite=Strict + Secure (en production)
Sécurité       : max 5 tentatives puis lockout 15 min (Netlify Blobs, côté serveur)
               : session expirée après 2h (JWT exp claim)
               : noindex/nofollow sur toutes les pages admin
               : headers HTTP : HSTS, X-Frame-Options, X-Content-Type-Options, etc.

Backend        : Netlify Functions (netlify/functions/)
  /api/login           — POST, vérifie mdp PBKDF2, retourne JWT cookie
  /api/logout          — POST, efface le cookie
  /api/me              — GET, vérifie le JWT (utilisé au chargement du dashboard)
  /api/site-data       — GET public / POST protégé JWT (données du site)
  /api/messages        — GET protégé JWT (messages de contact)
  /api/contact         — POST public, rate-limited 5/h par IP
  /api/change-password — POST protégé JWT + ancien mdp requis

Structure admin
  admin/index.html       — Page de login (appelle /api/login)
  admin/dashboard.html   — Dashboard complet (7 panneaux)
  admin/css/admin.css    — Thème sombre premium (variables indépendantes du site public)
  admin/js/admin.js      — Auth backend JWT, navigation, toasts, accordéons, CRUD portfolio

Panneaux disponibles
  Vue d'ensemble  — Résumé, accès rapide, toggles sections
  Hero / Accueil  — Titre, sous-titre, badge, boutons, cartes flottantes
  Services        — 4 cards éditables en accordéon + toggle visible/caché
  Statistiques    — 4 chiffres avec valeur + suffixe
  Portfolio       — 6 projets avec ajout/édition/suppression
  Contact         — Email, réseaux sociaux, textes
  Apparence       — Palette 6 couleurs, logo, hauteur nav
  Paramètres      — Nom site, tagline, SEO, changement mot de passe

Stockage       : Netlify Blobs (webcraft-site → données site, webcraft-admin → config/auth)

## À éviter absolument

- Couleurs flashy hors palette définie
- Font-weight < 300 ou polices fantaisie
- Sections toutes identiques (alterner clair / sombre / gradient)
- Design trop épuré / trop blanc sans contraste
- Animations lourdes (particules canvas, parallax intensif)
- Commentaires excessifs dans le code
- Templates génériques ou Bootstrap
